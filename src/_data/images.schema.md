# images.json — Field Schema

Each entry in `images.json` represents one published image. Fields:

| Field             | Type           | Required | Notes |
|-------------------|----------------|----------|-------|
| slug              | string         | yes      | URL key — must be unique, lowercase, kebab-case |
| title             | string         | yes      | Display name |
| catalog           | string or null | no       | Catalog designation(s), e.g. "NGC 1499 / IC 434" |
| tags              | string[]       | yes      | At least one tag. First tag drives the badge color on gallery cards. Valid values: emission-nebula, reflection-nebula, dark-nebula, planetary-nebula, supernova-remnant, galaxy, open-cluster, globular-cluster, solar, other |
| catalogs          | string[]       | no       | Membership collections for the catalog filter pill: "messier", "caldwell" |
| date              | string         | yes      | ISO date string "YYYY-MM-DD" |
| featured          | boolean        | no       | If true, shown in the home page hero/preview section |
| astrobin_id       | string or null | no       | AstroBin image ID (e.g. "d9vpf4") — renders a "View on AstroBin" link on the detail page |
| thumbnail         | string         | yes      | Path or URL to the 600px WebP thumbnail shown on gallery cards |
| preview_url       | string or null | no       | Path or URL to the 2400px WebP preview shown on the detail page hero |
| full_url          | string or null | no       | Path or URL to a full-size JPG (transitional; used when DZI tiles aren't ready) |
| annotated_url     | string or null | no       | Path or URL to an annotated JPG (fallback annotation swap for non-DZI images) |
| dzi_url           | string or null | no       | Full URL to the DZI descriptor on R2 — enables the deep-zoom lightbox |
| annotated_dzi_url | string or null | no       | Full URL to an annotated DZI on R2 — enables tile-swap annotations in the lightbox |
| equipment         | object or null | no       | { telescope, camera, mount, guider, filters, location, software } |
| acquisition       | object or null | no       | { mode?, note?, filters: [{name, frames, minutes}] } — minutes: null for lucky/solar imaging |
| sky               | object or null | no       | { ra, dec, fov_deg, aladin_target, ra_deg, dec_deg, fov_w, fov_h } — null for solar images |
| annotations       | object[]       | no       | [{name, x, y}] — x and y are 0–1 fractions from top-left corner of the image, used for OSD overlays |
| description       | string or null | no       | Long-form prose description of the image and capture details |

## Hero display priority (detail page)

1. `dzi_url` set → thumbnail preview + deep-zoom lightbox on click
2. `full_url` set → plain `<img>` (transitional)
3. Neither → placeholder div

## Preview image priority (detail page hero src)

1. `preview_url` — 2400px WebP (preferred; generated by the ingest pipeline)
2. `full_url`
3. `thumbnail` — 600px WebP (pixelated at hero size; add preview_url to fix)

## sky.fov_deg convention

`fov_deg` should equal `max(fov_w, fov_h)` — the widest dimension of the camera's
field of view. The Aladin Lite widget is initialized to this value so the FOV
rectangle fits within the sky atlas view without clipping.
